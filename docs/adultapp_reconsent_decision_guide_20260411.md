# 재동의 유예 종료 후 기능 제한 판단 가이드

## 이 기능이 뜻하는 것
필수 문서가 바뀌었을 때 기존 가입자에게 7일 유예를 주고, 그 기간이 끝난 뒤에도 새 문서에 동의하지 않으면 어떤 수준으로 기능을 제한할지를 정하는 정책입니다.

## 선택지
### 1. limited_access
- 로그인 허용
- 글쓰기, 채팅, 주문, 문의, 프로필 수정 제한
- 재동의 화면으로 먼저 이동
- 장점: 이용자 반발이 적고 공지/유예 정책과 잘 맞음
- 단점: 일부 읽기 전용 접근은 남음

### 2. login_block
- 로그인 자체 차단
- 재동의 완료 후에만 진입 가능
- 장점: 정책 강제력이 매우 높음
- 단점: CS 문의와 이탈이 늘 수 있음

## 현재 추천
초기 오픈은 `limited_access`가 적절합니다.
- 공지 후 재동의 유도에 유리
- 기능 쓰기 영역만 막아 법적/운영 리스크를 크게 줄일 수 있음
- 추후 필요 시 `login_block`으로 강화 가능
