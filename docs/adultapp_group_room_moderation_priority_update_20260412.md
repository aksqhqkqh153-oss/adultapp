## 질문사항 답변 정리
### 1) API 레벨에서 강제한다는 의미
- 프론트 화면에서 버튼을 숨기는 것만이 아니라, **서버 API 자체가 권한 없는 요청을 거절**해야 한다는 뜻.
- 예: 계정비공개면 상호 팔로우 아닌 사용자가 프로필/피드 API를 직접 호출해도 서버가 403 또는 제한 응답을 반환해야 함.
- 예: 관리자 로그는 관리자 권한 토큰으로만 응답해야 함.
- 즉 `계정별` + `권한별` 둘 다 체크해야 함.

### 2) 신고/차단/누적 제재 자동화에서 추가로 더 필요한 것
- 신고 접수 즉시 임시 블라인드 기준.
- 동일 대상 반복 요청 쿨다운.
- 차단 후 재요청 자동 금지.
- 동일 계정/기기/IP 반복 위반 누적 카운트.
- 일정 누적 시 채팅 제한, 단체 톡방 개설 제한, 1:1 요청 제한 자동 반영.
- 관리자 열람 로그 남기기.
- 신고 종결 전 대화 백업 보존 기간 기준.

### 3) localStorage 기반 기능을 계정 동기화한다는 의미
- 지금은 브라우저에만 저장되므로, 다른 기기/브라우저에서는 즐겨찾기/보관함이 안 보일 수 있음.
- 계정 동기화는 사용자가 로그인하면 서버 DB에 저장하고, 어느 기기에서 로그인해도 같은 즐겨찾기/보관함을 내려주는 구조를 뜻함.
- 즉 `내 PC에만 저장`이 아니라 `내 계정 자체에 저장`으로 바꾸는 것.

## 추가 판단/결심 필요 항목
1. 단체 톡방 개설을 성인인증만으로 계속 열어둘지, 나중에 신고 누적 기준을 더 강화할지.
2. 사진/영상/파일 금지는 계속 유지할지.
3. 앱 내부 공유 허용 범위를 주문/장바구니까지 넓게 유지할지.
4. 판매자 신청 항목 중 통신판매업 신고번호를 필수로 받을지, 승인 전 임시저장 허용할지.
5. SKU 정책에서 `보류` 품목을 얼마나 공격적으로 허용할지.
6. 프리미엄 배송 SLA를 판매자 자율 신청제로 둘지, 운영자 검수 통과 판매자만 허용할지.
7. 스토어 메타데이터를 현재처럼 보수적으로 유지할지.
