- 공개 베타 운영키 전환 기준을 `.env.example`, 백엔드 설정, API 응답 문구에 반영했습니다.
- 미성년 차단 계정은 `로그인 불가 및 접속 차단`, `최소 식별값과 차단 이력만 일정 기간 보관` 정책을 문서/프론트 안내에 반영했습니다.
- 최상위 관리자 예외는 `감사로그를 남기고 건별 승인만 허용` 원칙으로 설정과 문서에 반영했습니다.
- 재동의 7일 유예의 의미와 대상 사용자를 프론트 안내와 문서에 명확히 적었습니다.

1. PASS 실계약사 선택 후 실제 클라이언트 키/시크릿 연결
2. 콜백 실패 코드별 사용자 문구와 재시도 UX 세분화
3. 미성년 차단 계정 파기 시점 자동화
4. 관리자 예외 승인용 내부 화면 또는 승인 프로토콜
5. 재동의 유예 종료 후 기능 제한 범위 세부 확정

- 답변 예시: `예외 요청은 문서 또는 백오피스에서 등록하고, 승인 건마다 감사로그를 남긴다.`
