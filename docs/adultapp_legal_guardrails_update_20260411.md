# adultapp legal guardrails update 2026-04-11

## 반영 요약
- 회원가입 시 필수/선택 동의를 분리하고, 선택 미동의만으로는 가입이 막히지 않도록 백엔드 `ConsentRecord` 저장 로직을 추가함.
- `PASS/휴대폰 본인확인 시작/확인`, `성인인증 시작/확인`, `성인인증 상태` API를 추가함.
- 성인인증 실패 5회 기준 1시간 잠금 상태를 서버 필드(`adult_verification_fail_count`, `adult_verification_locked_until`)로 관리하도록 확장함.
- `auth/login`은 일반 계정이 본인확인 미완료 상태면 로그인 거절하도록 변경함. 관리자 계정은 예외 처리 유지.
- `auth/me`에 재동의 필요 여부, 랜덤채팅 필수 프로필 준비 여부, 성인인증 잠금 상태를 포함함.
- `legal/documents`, `legal/{doc}` API를 추가하고 가입 화면에서 이용약관/개인정보 처리방침/청소년 보호정책 고정 링크를 노출함.
- 비성인/비로그인 사용자가 상품/콘텐츠/커뮤니티 글 목록을 조회할 때 `safe` 가시성만 보이도록 필터를 추가함.
- 텍스트 사전 점검용 `POST /api/moderation/check-text` API를 추가함.
- 상단 검색/설정 아이콘은 테두리 없는 두툼한 스타일로 유지하고, 설정 아이콘은 일반적인 톱니형 SVG로 교체함.

## 이번 반영에서 남겨둔 운영 결심 포인트
1. 실제 본인확인 공급사 결정
   - 권장: PortOne + PASS/휴대폰 본인확인
   - 대안: 다날/KCP 직접 연동

2. 미성년 판정 시 정책
   - 권장: 계정은 유지 가능, 성인 기능 전체 차단
   - 강경안: 계정 자체 차단

3. 재동의 트리거 기준
   - 권장: 필수 문서 버전 변경 시 `auth/me.reconsent_required = true`

4. 랜덤채팅 필수 프로필 서버 차단 범위
   - 권장: `gender`, `age_band`, `region_code` 미입력 시 랜덤채팅 티켓/매칭 API도 거절

5. 불법 콘텐츠 대응 범위
   - 현재: 연락처/외부연락 유도 차단 중심
   - 추가 권장: 키워드 룰 + 관리자 검토 + 반복 위반 자동 제한
