# adultapp 본인확인/성인인증/법정문서 결정 항목

## 1. 인증사 확정
- 결정 필요: PASS 단독 / PASS+휴대폰 / 통합인증
- 권장 답변 예시: "1차 오픈은 PASS+휴대폰 본인확인으로 간다. 카카오는 로그인 편의 수단으로만 사용한다."

## 2. 운영키 전환 시점
- 결정 필요: 테스트 운영 후 즉시 전환 / 베타 종료 후 전환
- 권장 답변 예시: "클로즈드 베타 동안은 테스트키, 외부 공개 베타 직전에 운영키로 전환한다."

## 3. 미성년 처리 정책
- 결정 필요: 전체 계정 차단 / 계정 유지 + 성인 기능 차단
- 권장 답변 예시: "계정은 유지하되 홈 안전모드만 허용하고 성인 기능은 전부 차단한다."

## 4. 개인정보 처리방침 반영 공급사 범위
- 결정 필요: 실제 계약한 공급사만 기재 / 예정 공급사 범주까지 기재
- 권장 답변 예시: "정식 계약 완료 공급사만 확정 기재하고, 예정 항목은 화면 문구에 범주로만 안내한다."

## 5. 성인인증 실패 정책
- 결정 필요: 현재 5회/1시간 유지 / 더 강화 / 더 완화
- 권장 답변 예시: "현재 기준 5회 실패 시 1시간 쿨타임을 유지한다."

## 6. 관리자 예외 범위
- 결정 필요: 관리자 전면 예외 / 운영관리자 일부 예외 / 예외 없음
- 권장 답변 예시: "최상위 관리자만 예외를 두고, 일반 운영관리자는 동일 정책을 적용한다."

## 7. 법정문서 재동의 정책

- 권장 답변 예시: "필수 문서가 바뀌면 7일 유예 후 재동의를 강제한다."

## 8. 신고/차단 운영 강도
