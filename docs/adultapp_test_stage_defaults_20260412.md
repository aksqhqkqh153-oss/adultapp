# adultapp 테스트 단계 권장 설정값 (2026-04-12)

## 현재 단계 권장 결론

- PortOne 표준 SDK를 지금 붙입니다.
- test/live 결제 환경을 완전 분리합니다.
- SKU는 보수적으로 유지하고 허용 SKU만 테스트 상품으로 노출합니다.
- 프리미엄 배송 SLA는 보장형이 아니라 목표형/안내형으로 유지합니다.
- 판매자 필수 입력값 누락 시 상품 등록/공개/주문 수락을 차단하고, 테스트 관리자 계정만 override를 허용합니다.

## 이유

현재는 개발/테스트 단계이므로 운영 직전 재작업을 줄이기 위한 구조 고정이 더 중요합니다.

## 바로 다음 순서

1. PortOne 테스트 webhook secret 발급
2. test/live 환경변수 분리 입력
3. 테스트 결제/취소/부분취소/환불/webhook 검증
4. 판매자 필수 입력 누락 차단 동작 확인
5. 허용 SKU만 공개하여 PG 사전상담 진행
