# adultapp_order_payment_admin_test_flow_update_20260412

## 반영 내용
- 쇼핑 > 주문 탭을 선택 주문 기준 결제 테스트 센터로 보완
- 주문 목록에서 테스트 대상 주문을 직접 선택할 수 있게 변경
- 주문 상세 조회 API(`/api/orders/{order_no}`) 추가
- 선택 주문 기준 결제승인, 전체취소, 부분취소, 전체환불, 부분환불, webhook 점검 동선 보완
- 선택 주문 상세에서 금액 스냅샷, payment record, 결제 history를 확인 가능하게 추가
- 관리자 상품등록 후 일반 회원 목록 노출 확인 흐름을 유지하면서 상품 등록 완료 메시지 보완

## 회원 계정 테스트 경로
1. 회원 로그인
2. 쇼핑 > 목록 > 상품 선택
3. 장바구니 담기
4. 쇼핑 > 바구니 > 주문 생성
5. 쇼핑 > 주문 > 생성된 주문 `주문선택`
6. 선택 주문 결제승인
7. 선택 주문 부분취소 / 전체취소
8. 선택 주문 부분환불 / 전체환불
9. webhook 점검
10. 주문 상세 / 결제 스냅샷 확인

## 관리자 계정 테스트 경로
1. 관리자 로그인
2. 쇼핑 > 상품등록
3. 테스트 상품 등록
4. 일반 회원 계정에서 쇼핑 목록 노출 확인
5. 쇼핑 > 주문에서 주문 상태 변화 확인

## 빌드 점검
- frontend: `npm run build` 통과
- backend: `python -m compileall backend/app` 통과
