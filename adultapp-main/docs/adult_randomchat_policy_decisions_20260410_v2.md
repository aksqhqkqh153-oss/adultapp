# adult randomchat policy decisions - 2026-04-10 v2

## 이번 확정 반영
- 거리 점수식: `band_bonus` 구간형 가점
- 연령 인접 허용: `30대 ↔ 40대`
- 차단 해제 권한: 일반 사용자 자가 해제 허용 + 관리자 로그 무조건 기록
- 영구정지 후 기존 채팅: 읽기전용 유지 + 관리자 전용 보관소 동시 저장
- 메시지 삭제: 사용자 화면에는 `메세지가 삭제되었습니다` 표기, 원문은 관리자 전용 보관소 저장
- 지역 공개 수준: `시` 단위만 공개
- 지역 표기 표준: `서울특별시→서울`, `인천광역시→인천` 등 축약 표준 사용
- 신고 집계 기준: 누적 건수가 아니라 `누적 점수` 기준
- 신고 가중치: 욕설 1점 / 스팸 1점 / 개인정보요구 2점 / 음란물전송 3점 / 불법권유 3점 / 기타 1점
- 자동정지 기준: `5점=3일`, `10점=7일`, `20점=30일`, `21점 이상=관리자 검토`
- 운영 DB: Railway PostgreSQL 고정, SQLite는 로컬 전용

## 보관 기간 운영안
- 삭제 메시지 원문, 신고 로그, 영구정지 증거는 기본 `3년` 분리보관
- 주문/결제/환불 분쟁과 직접 연결된 자료는 전자상거래 기록 보존 기준에 맞춰 `5년`까지 연장 가능
- 법무 대응, 수사 협조, 소송 홀드가 걸린 자료는 별도 `legal_hold` 상태로 만료 정지
- 보관 만료 후에는 복구 불가 방식으로 파기

## WebSocket 확장 기준 확정
- 초기 운영: Railway 단독 단일 인스턴스 유지
- Redis 전환 시점:
  - WebSocket 동시 접속이 단일 인스턴스 기준을 지속 초과
  - 채팅 서버를 2개 이상 인스턴스로 늘려야 함
  - 메시지 유실/지연이 반복되거나 배포 중 소켓 끊김 영향이 커짐
  - 운영 메모리/CPU가 상시 고부하로 유지됨
- 전환 구조: Railway App + Redis Pub/Sub + sticky session 최소화

## 질문사항 의미 정리
### 1) 읽기전용 스레드 범위가 무슨 말인가
영구정지된 상대와의 예전 채팅방을 `어디까지 보이게 할지`를 뜻합니다.
- 프로필 허용 범위: 닉네임/연령대/지역축약까지만 계속 보여줄지
- 첨부 허용 범위: 기존 이미지/영상 썸네일만 볼지, 원본 열람도 허용할지
- 재접속 허용 범위: 예전 방 목록 재진입은 허용하되 새 메시지는 금지할지

### 2) 관리자 보관소 권한이 무슨 말인가
삭제 메시지 원문, 신고 증거, 영구정지 기록을 `누가 볼 수 있는지` 정하는 것입니다.
- 단순안: `grade 1` 관리자만 열람 가능
- 분리안: `grade 1`은 전체 권한, `감사/모더레이션 전용 보조권한`은 열람만 가능
- 권장: 초기에는 `grade 1만 열람`, 추후 운영 인력이 늘면 `감사 전용 읽기권한` 추가

## 추가로 결심 필요한 항목
### A. 고유 신고자 중복 처리
- 결정: 같은 사용자가 같은 상대를 반복 신고해도 `1회만 점수 반영`할지 여부
- 권장: 최근 24시간 동일 신고자는 1회만 반영

### B. 허위 신고 패널티
- 결정: 악의적 반복 허위 신고자 제재 여부
- 권장: 반려 3회 누적 시 경고, 5회 누적 시 신고권 일시 제한

### C. 차단 해제 쿨다운
- 결정: 차단 후 즉시 해제 허용 여부
- 권장: 차단 후 10분~1시간 쿨다운 부여

### D. 메시지 삭제 시점
- 결정: 발송 후 몇 분까지 사용자 자가 삭제 허용할지
- 권장: 5분 이내 자가 삭제 허용, 이후 관리자 요청 처리

### E. 첨부파일 정책
- 결정: 랜덤채팅에서 이미지/영상 허용 범위
- 권장: 초기에는 텍스트 우선, 첨부는 이미지 1장 이하 또는 비활성

### F. 프로필 노출 최소화
- 결정: 매칭 전/후 보여줄 프로필 범위
- 권장: 닉네임, 성별, 연령대, 지역축약만 표시

### G. 관리자 조치 이력 통지
- 결정: 상대 신고 처리결과를 신고자에게 어느 수준까지 통지할지
- 권장: `조치 완료/검토 중` 수준만 회신하고 상세 제재 기간은 비공개
