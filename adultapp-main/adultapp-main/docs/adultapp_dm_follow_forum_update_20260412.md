# adultapp DM + follow + forum update (2026-04-12)

## 적용 내용
- 채팅 > 채팅: 1:1 채팅은 요청 → 수락 → 대화 시작 2단계 구조 유지
- 1:1 요청 전 대화 규칙 동의 체크 UI 반영
- 1:1 채팅 상단 운영 기준 안내 반영
- 1:1 채팅에서 사진/영상/파일 전송 금지 유지
- 상호 팔로우 상태에서만 1:1 요청 가능하도록 UI/정책 반영
- 채팅 > 포럼: 주제 버튼(제품 이야기, 경계설정 이야기, 초보 고민, 일상 대화, 성향 고민) 기반의 1:1 시작점 반영
- 백엔드: mutual follow 전용 1:1 요청/수락 라우트 및 정책 요약/팔로우 요약 라우트 추가

## 이번 반영의 핵심 포인트
- 자유 DM이 아니라 상호 팔로우 + 요청/수락형 1:1 구조로 제한
- 외부 연락처 교환, 사진/영상 전송, 반복 접촉, 오프라인 만남 제안 금지 문구를 명시적으로 고지
- 포럼 기반의 정보/상담형 대화에서 1:1로 이어지는 흐름을 앱 안에서 보여주되, 매칭/만남 서비스처럼 보이지 않도록 구조를 조정

## 추가 조치 필요
1. 백엔드 인증 연동
- 현재 UI/라우트는 반영되었지만, 실제 운영 전에는 현재 로그인 사용자와 실제 팔로우 관계를 DB/API 기준으로 연결해 테스트 필요

2. 신고/차단 후속 처리 고도화
- 반복 접촉, 차단 후 재요청, 허위 신고, 누적 위반 점수 정책을 실제 운영 데이터 기준으로 보강 필요

3. 스토어/법률 문구 정리
- 앱 설명, 카테고리명, 채팅방 설명에서 `매칭`, `만남`, `파트너 찾기`처럼 읽히는 표현은 계속 피하는 편이 안전

4. 운영자 판단 필요
- `성향 고민` 주제 버튼은 유지 가능하지만, 앱스토어 심사나 민원 대응에서 민감하게 읽힐 수 있으므로
  - 유지할지
  - `관계 역할 고민` 등 더 중립적인 문구로 바꿀지
  최종 결정 필요

5. 실제 운영 전 필수 검토
- 변호사 검토: 약관, 청소년보호정책, 개인정보처리방침, 채팅 운영정책
- 내부 운영정책: 채팅 캡처 증거 제출, 관리자 블라인드, 제재 수위, 재심 절차

## 현재 판단
- 이번 구조는 이전의 자유 1:1보다 훨씬 안전한 방향
- 다만 `상호 팔로우 후 1:1`도 운영이 느슨하면 외부 이동/오프 제안으로 번질 수 있으므로, 운영정책과 신고처리가 핵심
